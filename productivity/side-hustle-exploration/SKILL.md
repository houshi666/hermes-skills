# Side Hustle Exploration Guide

Help users overcome analysis paralysis and move from "too many ideas" to "taking action" when exploring side hustles or career second curves.

## Trigger Conditions
- User expresses anxiety about career direction or side hustle choices
- User has many ideas but can't commit to one
- User is in "research mode" but never taking action
- User says things like "不知道做什么" / "很迷茫" / "想法很多但无法落地"

## Core Approach

### 1. Identify the Real Problem
Most often it's not "no options" but **analysis paralysis**:
- Too many success stories → fear of picking wrong
- Research feels productive → but never leads to action
- Perfectionism → won't start until "ready"

### 2. Find Their Core Competency Pattern
Ask: "在过去的工作/生活中，有哪些事情是你**自发做过且帮到别人的**？"

Look for patterns in what they've already done:
- Technical problem-solving?
- Content creation?
- Consulting/advising?
- Product building?

The best side hustle often hides in **what they already do naturally**.

### 3. Use Exclusion Method, Not Selection Method
Instead of "哪个最好" → ask "哪个我最不抗拒开始"

Create a comparison table:
| 方向 | 启动成本 | 变现周期 | 竞争程度 | 与当前工作关联 |
|------|---------|---------|---------|---------------|

### 4. Propose Minimum Viable Test
Key principle: **"先让轮子转起来"** (Let the wheel turn first)

- Pick the lowest-barrier option
- Design a 1-2 week test with clear success criteria
- Success = getting real feedback, NOT making money
- Frame it as "experiment" not "commitment"

### 5. Break Into Ultra-Small Steps
Daily tasks should be <1 hour:
| 时间 | 任务 | 耗时 |
|------|------|------|
| Day 1 | Small prep task | 15-30 min |
| Day 2 | Create something | 30 min |
| Day 3 | Polish + publish | 20 min |
| Weekend | Launch + observe | 随意 |

### 6. Validate Before Scaling
Before investing serious time:
- Test in low-cost channels first (朋友圈，前同事群)
- Look for organic interest (私信，咨询)
- If no response after 2 weeks → pivot, don't persist

## Common Pitfalls

| Pitfall | Solution |
|---------|----------|
| "竞争太激烈" | 细分定位，服务特定人群 |
| "还没准备好" | 完成比完美重要，先发布再迭代 |
| "万一失败了怎么办" | 定义为"测试"不是"承诺"，2 周无反馈就换方向 |
| "这个方向真的好吗" | 真实反馈比任何分析都有价值 |

## Key Phrases to Use
- "不是选对方向，而是先让轮子转起来"
- "用排除法而不是选择法"
- "启动成本最低的是哪个？"
- "2 周测试，无反馈就换，没损失"
- "方向不在外面，在你已经做过且享受的事里"

## Success Metrics
- User commits to a specific first step
- User has a concrete timeline (not "someday")
- User shifts from "which is best" to "let me try this"

## Notes
- Don't push too hard — respect user's pace
- Some users need to vent before they're ready to act
- The goal is momentum, not perfection
- Connect to existing work experience when possible (lowest friction)
